pub struct Channel { /* private fields */ }
Expand description

Representation of an APB DMA Channel.

Channels are used for data transfers over DMA by the DMA Controller and need to be acquired and released when using them to avoid data races.

NOTE: Instances of this struct should never be created manually. Refer to the public constants this struct holds, which represent the channels 0 through 31.

pub fn acquire(&mut self) -> Result<(), ()>

Acquires the selected channel.

Returns an error if the channel is already claimed at the time of calling the method. When the channel is not needed anymore, it must be freed with Channel::release.

source

pub fn release(&mut self) -> Result<(), ()>

Releases the selected channel after it is not needed anymore.

Returns an error if the channel is not claimed at the time of calling the method.

source

pub fn is_acquired(&self) -> bool

Indicates whether the channel is currently claimed.

source

pub fn is_free(&self) -> bool

Indicates whether the selected channel is currently unused.

This is the counterpart to Channel::is_acquired.

source

pub fn start(&self)

Starts a DMA transfer in the selected channel.

source

pub fn finish(&self)

Finishes a DMA transfer in the selected channel.

source

pub fn is_busy(&self) -> bool

Indicates whether the channel is currently busy doing transfers.

NOTE: Channels need to be acquired before being able to transfer data. If this method returns true, Channel::is_acquired is guaranteed to return true as well.

source

pub fn is_idle(&self) -> bool

Indicates whether the channel is in idle state and ready for transfers.

This is the counterpart to Channel::is_busy.

source

pub fn query(
    &self,
    slave: u32,
    ahb_address: u32,
    apb_address: u32,
    size: u32
) -> Result<(), ()>

Prepares data to be queried through the selected channel over DMA.

This function doesn’t actually trigger transfers, it only prepares the data output buffer and configures the channel. The actual transfer process can be initiated through Channel::start and terminated through Channel::finish. For transfer status details, see Channel::is_busy.

pub fn write(

    slave: u32,
    ahb_address: u32,
    apb_address: u32,
    size: u32
) -> Result<(), ()>

Prepares data to be written through the selected channel over DMA.

This method doesn’t actually trigger transfers, it only loads in the data and configures the channel. The actual transfer process can be initiated through Channel::start and terminated through Channel::finish. For transfer status details, see Channel::is_busy.
